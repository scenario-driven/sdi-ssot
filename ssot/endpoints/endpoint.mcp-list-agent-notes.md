## 정의

`list_agent_notes` 는 SDI MCP 서버의 읽기 도구로, LLM 에이전트가 M1 블랙보드 패턴에 따라 다른 에이전트가 남긴 노트를 조회할 때 사용한다. 데몬의 `GET /agent_notes` 엔드포인트로 위임된다.

M1 블랙보드는 여러 에이전트가 비동기적으로 협업할 때 중앙 공유 칠판 역할을 한다. 한 에이전트가 작업을 완료하거나 다음 에이전트에게 맥락을 전달할 때 노트를 남기면, 수신 에이전트가 이 도구로 자신에게 전달된 노트를 꺼내 읽는 방식이다. 이 도구는 두 가지 조회 모드를 지원한다: 특정 에이전트에게 전달된 핸드오프를 찾는 수신 모드와, 특정 플랜·태스크·라운드에 부착된 노트 전체를 보는 스코프 모드.

## 요청 / 응답

**입력이 의미하는 것**

조회 모드에 따라 입력이 달라진다.

수신 모드에서는 조회 대상 에이전트의 이름을 지정한다. "나에게 온 핸드오프가 있는가"를 확인하는 용도로, 에이전트가 작업 세션을 시작할 때 가장 먼저 호출하는 패턴이다.

스코프 모드에서는 스코프 종류(plan/task/round 중 하나)와 해당 엔티티의 ID를 지정한다. "이 태스크와 관련하여 다른 에이전트들이 무엇을 기록했는가"를 파악할 때 사용한다.

**출력이 의미하는 것**

응답은 에이전트 노트 항목의 배열이다. 각 항목은 다음 정보를 포함한다. 작성 에이전트는 이 노트를 남긴 에이전트의 식별자다. 내용은 노트의 실제 텍스트로, 관찰 사항, 수행한 작업 요약, 다음 에이전트에 대한 지침, 또는 이상 감지 내용이 담긴다. 종류는 이 노트의 성격을 나타낸다: `observation`(관찰), `handoff`(작업 인계), `directive`(지침), `summary`(완료 요약). 상태는 이 노트가 아직 확인되지 않았는지(`pending`), 수신 에이전트가 확인했는지(`ack`), 더 이상 유효하지 않은지(`retired`)를 나타낸다. 생성일시는 이 노트가 언제 작성되었는지다.

## 권한 / 제약

이 도구는 읽기 전용이며 노트의 상태를 변경하지 않는다. `pending` 노트를 읽어도 자동으로 `ack` 상태로 전환되지 않는다 — 상태 전환은 별도 메커니즘이 필요하다(미확인).

수신 모드에서는 `to_agent` 필드에 지정된 에이전트 이름과 정확히 일치하는 핸드오프 노트만 반환된다. 이름 기반 필터이므로 에이전트 식별자가 일관되게 사용되어야 한다.

스코프 모드에서는 앵커 엔티티 ID가 반드시 필요하다. 스코프 없이 전체 블랙보드를 조회하는 것은 이 도구로 불가능하다.

## 미확정 (OPEN)

- `pending` 노트 조회 후 `ack` 상태 전환이 이 도구 내에서 자동으로 일어나는지, 아니면 별도 쓰기 도구를 호출해야 하는지 미확인
- 노트 작성(`add_agent_note`) 도구가 현재 MCP 도구 목록에 별도로 존재하지 않는 것으로 파악됨 — 블랙보드 쓰기 경로가 어디에 있는지 미확인
- `retired` 상태 노트가 응답에 기본적으로 포함되는지, 필터 옵션이 있는지 불명확
- 두 모드(수신/스코프)를 하나의 도구 호출에서 혼합 사용하는 것이 가능한지 미확인
